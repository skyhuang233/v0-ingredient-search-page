# Bio-Adaptive Agentic RAG Smart Search
> **Feature Documentation & Developer Guide**

## 1. 系统概述 (System Overview)

本系统实现了一个**生物自适应的智能搜索功能 (Bio-Adaptive Agentic RAG)**。不同于传统的关键词匹配，本系统结合了**向量检索 (Vector Search)** 与 **大模型决策 (LLM Agentic Reranking)**，能够根据用户的生理特征（如增肌、减脂需求）智能重排食谱，并提供营养师级的推荐理由。

### 核心架构 (Architecture)

系统采用 **双阶段 RAG (Two-Stage Retrieval-Augmented Generation)** 架构：

1.  **Stage 1: 广度检索 (Broad Retrieval)**
    *   **组件**: `DenseRecommender` (Python/Faiss)
    *   **行为**: 基于食材的语义向量，快速召回 Top-20 相关食谱。
    *   **目的**: 确保候选集覆盖面广，不漏掉潜在匹配。

2.  **Stage 2: 智能重排 (Agentic Reranking)**
    *   **组件**: `Agent Reranker` (TypeScript/DeepSeek V3)
    *   **行为**: 
        *   注入用户画像 (User Profile: 目标/体重/偏好)。
        *   注入候选食谱上下文 (Recipe Context)。
        *   LLM 进行推理，筛选出最符合用户健康目标的 Top-3 食谱。
        *   生成 "Nutritionist Note" (营养师建议) 和 "Reasoning" (推荐理由)。

## 2. 快速开始 (Getting Started)

### 环境要求
*   Node.js 18+
*   Python 3.10+ (后端推荐服务)
*   **DeepSeek API Key**

### 配置步骤 (Configuration)

1.  **配置环境变量**
    在项目根目录 `.env.local` 文件中添加：
    ```bash
    # LLM Service Provider
    DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
    
    # Backend Service URL (Default)
    RECOMMENDER_URL=http://localhost:8000/recommend
    ```

2.  **启动服务**
    确保 Python 后端与 Next.js 前端均处于运行状态。
    ```bash
    # Terminal 1: Frontend
    npm run dev
    
    # Terminal 2: Backend (if running locally)
    python -m recommender.main
    ```

## 3. 开发指南 (Developer Guide)

本模块遵循**高内聚、低耦合**的开发原则，核心逻辑封装在独立模块中。

### 目录结构 (Directory Structure)

```text
e:\v0-ingredient-search-page\
├── app\
│   ├── api\recommend\route.ts   # [API] 编排层：聚合用户数据 -> 调用检索 -> 调用重排
│   └── search\page.tsx          # [UI] 展示层：渲染食谱及营养师建议
├── lib\
│   └── agent-reranker.ts        # [Core] 核心逻辑：封装 LLM Client 与 Prompt
└── ...
```

### 核心代码说明

#### 1. Agent Reranker (`lib/agent-reranker.ts`)
这是智能体的核心大脑。
*   **Prompt Engineering**: 提示词模板位于 `rerankRecipes` 函数中。
*   **Model Swapping**: 目前配置为 `deepseek-chat`。如需切换模型（如 GPT-4o），修改 `baseURL` 和 `model` 参数即可。
*   **Structured Output**: 强制 LLM 输出 JSON 格式，确保系统稳定性。

**如何修改提示词 (Prompt)?**
找到 `systemPrompt` 变量：
```typescript
const systemPrompt = `
  You are an expert Nutritionist...
  // 修改这里的 Instruction 可以调整 AI 的语气或筛选逻辑
`
```

#### 2. API Route (`app/api/recommend/route.ts`)
流量入口与编排器。
*   负责获取当前用户 ID (UserId)。
*   负责从 Supabase 拉取用户画像 (Profiles)。
*   **注意**: 如果为了测试目的需要 Mock 用户数据，可以在这里手动构造 `userProfile` 对象。

## 4. API 接口文档 (API Reference)

### `POST /api/recommend`

**请求参数 (Request Body)**
```json
{
  "ingredients": ["chicken", "broccoli"], // 用户输入的食材
  "userId": "user_123",                   // (可选) 用户ID，用于拉取画像
```

```json
{
  "results": [
    {
      "recipe_id": 101,
      "name": "Chicken Salad",

      // ...其他原有字段
    }
  ],
  "nutritionist_note": "Given your goal of weight loss..." // Agent 生成的全局建议
}
```

## 5. 协作规范 (Collaboration Standards)

1.  **Prompt Versioning**: 修改 `lib/agent-reranker.ts` 中的 Prompt 时，建议在注释中记录修改原因和版本 (e.g., `// v1.1 Added strictly JSON constraint`)。
2.  **Error Handling**: LLM 服务可能超时或失败。`rerankRecipes` 内置了 Try-Catch 机制，失败时会**自动降级 (Fallback)** 为原始检索结果，确保用户体验不受阻断。
3.  **Type Safety**: 保持 `ExtendedHealthProfile` 和 `RerankedResult` 接口定义的准确性。
